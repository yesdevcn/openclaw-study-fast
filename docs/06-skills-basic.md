# 第 6 章：Skills 入门——给 OpenClaw 装上「超能力」

如果说 OpenClaw 是「大脑」，那 **Skills 就是它的手脚和工具箱**。

本章重点回答三件事：

- Skills 到底是什么，有什么用？
- 怎么安装 / 启用 / 管理 Skills？
- 新手一开始应该优先玩哪几类 Skills？

---

## 6.1 用一个比喻理解 Skills

你可以把 Skills 想象成手机里的「App」：

- OpenClaw 是操作系统；
- 大模型是「系统自带的大脑」；
- Skills 是一个个 App，负责具体能力，比如：
  - 管理文件；
  - 读写日历、待办；
  - 抓网页、调 API；
  - 连接飞书/企微/钉钉/QQ 等。

不同之处在于：

- Skills **主要是给 AI 用的**，不是给人点按钮；
- 多个 Skills 可以组合在一个任务里一起用；
- 官方 + 社区一起维护，生态会非常多样。

---

## 6.2 Skills 一般能做什么？

按用途大致可以分为几类（只列常见的）：

- **文件 & 知识类**
  - 读写本地/云端文件；
  - 遍历目录、搜索内容；
  - 管理个人知识库。

- **时间 & 日程类**
  - 创建/修改/查看日历事件；
  - 管理待办、提醒事项。

- **网络 & API 类**
  - 抓取网页内容；
  - 调用第三方 HTTP API；
  - 和 Notion、GitHub 等服务打通。

- **自动化 & 调度类**
  - 定时执行任务；
  - 监控网站/接口变化并提醒你。

- **IM 集成类**
  - 把 OpenClaw 接入飞书、企微、钉钉、QQ 等；
  - 以机器人身份在群里工作。

你可以简单记住：**凡是你经常重复做、又希望 AI 替你干的活，基本都可以期待通过某种 Skill 来解决。**

---

## 6.3 在哪里发现和查看 Skills？

具体界面随版本可能略有差异，但通常：

- 在 Control UI 里能找到一个「Skills / 扩展 / 插件」类似的页面；
- 在配置文件（如 `openclaw.json`）里有一个 `skills` 配置段；
- 社区会有合辑仓库，例如 `awesome-openclaw-skills` 一类的项目。

这些地方一般会告诉你：

- Skill 名称和简介；
- 能力说明；
- 安装方式（UI/命令行/配置文件）；
- 所需的额外配置（如 API Key、服务地址等）。

---

## 6.4 安装与启用 Skills 的通用步骤

不同 Skill 细节不同，但「套路」相似：

1. **挑一个要用的 Skill**
   - 在控制台技能列表里搜索；
   - 或在教程/仓库里看到推荐。

2. **看清说明**
   - 有没有前置依赖（例如某云服务账号、API Key）；
   - 是否涉及安全敏感权限（读写文件、执行命令等）。

3. **执行安装**
   - 有的在 UI 里点一下「安装」；
   - 有的需要你在 `openclaw.json` 里增加配置；
   - 也可能提供 `openclaw` 命令行辅助安装。

4. **填配置**
   - 比如 API Key、默认目录、默认日历名称等；
   - 优先照抄官方或教程给的「最小可用配置」。

5. **在会话中试用**
   - 用自然语言发布送一个与该 Skill 相关的任务；
   - 如果失败，看错误信息，再回来微调配置。

---

## 6.5 新手优先推荐的 Skills 类型

一开始不用装太多，建议优先搞定这几类：

- **文件管理类**
  - 让 OpenClaw 能够列目录、读写文件；
  - 支撑「整理文件」「批量处理文档」等场景。

- **网页抓取类**
  - 给它一个链接，它能把网页内容抓回来总结；
  - 是搭建个人知识库的基础。

- **日历 / 待办类**
  - 用来把聊天内容、截图通知，变成真正的日历事件或待办；
  - 对「生活/工作安排」提升很明显。

- **通知/IM 集成类**
  - 把执行结果发到飞书/企微/钉钉/QQ；
  - 是自动化任务的常见终点（发日报、提醒等）。

等你在这几类上有了真实收获，再慢慢尝试更复杂的技能组合。

---

## 6.6 和 Skills 相处的几个小习惯

- **多描述任务，少纠结指令**
  - 你只需说清楚「想要什么结果」；
  - 具体用哪个 Skill，让 OpenClaw 自己选。

- **遇到错误，先读提示**
  - 很多时候是 Key 写错、权限不足、路径配置不对；
  - 可以直接把错误消息贴给 OpenClaw，请它帮你翻译和给建议。

- **不要一口气装太多**
  - 先少量、稳定，再逐步扩展；
  - 定期回顾：哪几个 Skills 真正在帮你省时间，其它可以暂时关掉。

---

## 6.7 展望：从「会用」到「会写」

对大多数人来说，**会用别人写好的 Skills 就足够产生巨大价值**。  
如果你是开发者，未来还可以：

- 学习官方给出的 Skill 标准和开发文档；
- 把你自己或团队的业务流程封装进 Skill；
- 发布到社区，反向获得用户反馈和改进建议。

本书的进阶章节会给出一两个「自定义 Skill」的思路和链接，让你有路可走。
