# 第 3 章：10 分钟安装与基础配置

本章的目标很单纯：**先把 OpenClaw 跑起来**，其他高级玩法后面再说。

我们会分两部分讲：

- 一条通用的「官方脚本安装路径」；
- 两个重点场景的额外说明：Mac 本地、云端服务器。

> 提示：本章示例命令可能随着官方版本略有变化，细节以官方文档为准，你可以把本章当作「中文导览 + 实战补充」。官方教程参考：`https://docs.openclaw.ai`。

---

## 3.1 环境准备

推荐环境（获得相对顺滑的体验）：

- **操作系统**：
  - 优先：macOS 12+（Apple Silicon 更佳）
  - 可用：Linux（Ubuntu / Debian / CentOS 等主流发行版）
  - 也可：Windows 10+（建议配合 WSL2）
- **基础工具**：
  - `curl` 或 `wget`
  - `git`
  - 一个现代浏览器（Chrome / Edge / Safari）

此外，你还需要：

- 至少一个可用的大模型 API Key（可来自 OpenAI、Anthropic 或国产模型/中转服务）；
- 能访问 `https://openclaw.ai` 和主流 API 服务的网络环境。

---

## 3.2 使用官方脚本安装 OpenClaw

下面是一条典型的安装命令（在终端中执行）：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

这条命令会帮你完成：

- 下载并安装 OpenClaw 核心组件；
- 在你的用户目录（通常是 `~/.openclaw/`）下创建必要的配置与数据目录；
- 安装基础命令行工具 `openclaw`。

安装成功后，你可以运行：

```bash
openclaw --version
```

如果能看到类似 `openclaw 2026.3.x` 的输出，说明安装已经就绪。

---

## 3.3 第一次启动 Gateway 与控制台

通常你可以用下面的命令启动 Gateway（网关服务）：

```bash
openclaw gateway start
```

启动成功后，终端或日志会提示一个访问地址，例如：

- `http://127.0.0.1:18789`（本地）
- 或者是你云服务器的 IP / 域名 + 端口

在浏览器中打开这个地址，你应该能看到 OpenClaw 的控制界面（Control UI）。  
第一次打开时，可能会要求你进行一些初始化设置（比如认证方式、基础配置等），按照向导一步步完成即可。

如果你只是想**先确认跑得起来**，可以暂时使用默认或自动生成的设置，后续再精细化配置。

---

## 3.4 最小可用配置：先让它能聊天

在所有高级功能之前，**先保证聊天正常、模型可用**是最关键的一步。

一般来说，你需要：

1. 在配置里填写至少一个模型提供商的 API Key；
2. 指定一个默认模型，用于日常对话。

假设你使用的是某个中转服务或国产模型提供商，通常需要在 `~/.openclaw/openclaw.json` 中写入类似结构（伪代码示例，仅作参考）：

```jsonc
{
  "models": {
    "providers": {
      "your-provider-id": {
        "base_url": "https://api.example.com/v1",
        "api_key": "YOUR_API_KEY",
        "type": "openai-compatible"
      }
    },
    "default": {
      "provider": "your-provider-id",
      "model": "gpt-4.1-mini" // 或国产模型名称
    }
  }
}
```

配置完之后，重启 Gateway：

```bash
openclaw gateway restart
```

然后在 Control UI 里新建一个会话，试着提几个问题：

- 「请用简体中文介绍一下你自己」；
- 「帮我列三件今天最重要的工作，并给出安排顺序」。

如果回答流畅、没有模型相关报错，说明基础模型配置已经成功。

## 3.5 Mac 本地部署的特别说明

如果你是在 macOS 上本地部署，有几个额外好处与注意点：

- 利用系统权限，OpenClaw 可以：
  - 访问本地文件系统；
  - 调用日历、提醒事项、通讯录等（根据官方内置 Skills 能力与授权情况）；
- 你可以更放心地把敏感资料只存放在本机，避免上云。

同时也有一些建议：

- 尽量使用较新的系统版本（例如 macOS 12+），兼容性更好；
- 给 OpenClaw 相关程序授予必要的系统权限（比如「完全磁盘访问」「日历访问」等）；
- 如果你使用的是代理 / 科学上网工具，确保终端环境与浏览器都能正常访问所需 API。

本书后续在讲「文件管理」「日程管理」等章节节时，会默认你已经在本地（或云端）成功跑起来，并有权限访问相关资源。

## 3.6 云端部署的特别说明

在云服务器上部署时，除了正常安装，还有几个「中国用户比较容易踩的坑」：

- **端口与安全组**：
  - 确保 OpenClaw 使用的端口（例如 18789）在安全组、防火墙中已放行；
  - 建议只对固定 IP 或通过 VPN/ZeroTrust 等方式进行访问控制。

- **域名与 HTTPS（可选但推荐）**：
  - 给你的网关配置一个域名，可以大幅简化访问体验；
  - 使用反向代理（如 Nginx / Caddy）为 Control UI 加上 HTTPS，提高安全性；
  - 很多浏览器在涉及敏感权限时，更倾向于信任 HTTPS 站点。

- **成本控制**：
  - 选择 2 核 4G + 轻量应用服务器即可满足个人/小团队使用；
  - 模型尽量优先选择国产或中转服务，将 Token 成本控制在可接受范围内；
  - 若只在工作时间使用，可考虑配合关机/睡眠策略进一步节省云服务器费用。

## 3.7 安装失败怎么办？

如果你在安装或首次启动时遇到了问题，可以按下面的思路排查：

1. **看错误信息**：
   - 安装脚本输出中是否有明显的「Permission denied」「Network error」「Command not found」等提示？

2. **检查网络环境**：
   - 能否正常访问 `https://openclaw.ai` 和你的模型提供商 API？

3. **查看日志 / 终端输出**：
   - Gateway 启动失败时，通常会在终端中打印原因；
   - 也可以在配置目录中找到对应的日志文件（路径以官方文档为准）。

4. **对照官方 FAQ 与社区教程**：
   - 本书会在第 8 章给出一个「常见错误 → 中文解释 → 排查步骤 → 官方链接」的对照表；
   - 你也可以查看社区维护的教程仓库，例如：[awesome-openclaw-tutorial](https://github.com/xianyu110/awesome-openclaw-tutorial)。

如果仍然搞不定，建议：

- 在 Issue 中附上：
  - 操作系统类型与版本；
  - 执行的命令；
  - 完整或关键的错误输出；
- 避免只发一句「安装失败了怎么办」，那样别人很难帮到你。
