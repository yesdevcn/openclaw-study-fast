# 第 5 章：三个高频实战场景（文件 / 知识 / 日程）🛠️

从这一章开始，我们不再停留在概念上，而是直接上手做三件事：

- 让 OpenClaw 帮你**管理本地/云端文件** 📂；
- 帮你搭一个**轻量的个人知识库** 📚；
- 让「截图/文案 → 自动变成日程和提醒」🗓️。

这些都是中国用户日常最常见、也最容易马上感受到价值的场景。

> 提示：本章示例会尽量保持「模型与提供商无关」，你只需要确保：OpenClaw 已经能正常调用基础文件、日历等内置 Skills（具体以官方版本为准）。

---

## 5.1 场景一：从一堆截图里找到发布票

### 5.1.1 典型问题

你可能会遇到这样的场景：

- 硬盘 / 手机里有一大堆聊天截图、发布票照片；
- 每次报销、记账时，都要一个个翻，非常浪费时间；
- 很多都是「长得差不多」的图片，很难肉眼快速区分。

### 5.1.2 基本思路

思路是：

- 把这一堆图片放到一个目录里（本地或云端挂载）；
- 让 OpenClaw 使用内置 / 配置好的文件与图像相关 Skills 去扫描；
- 按「时间」「金额」「商家」等条件帮你筛选出候选图片；
- 最后你只需要做人工确认。

### 5.1.3 示例工作流（伪代码级）

> 具体的 Skill 名称和调用方式可能会随版本变化，以下为思路演示。

1. 在会话中说明背景与目标：

```text
背景：我有一个目录，里面有很多图片文件，路径大致是：
- ~/Documents/receipts/2025-q4/

目标：请帮我从中找出「看起来像发布票或账单」的图片文件，并给出一个候选列表。

约束条件：
- 优先包含有金额、日期、商家名称的图片；
- 如果能识别出金额和日期，请一并列出。
```

2. 引导 OpenClaw 使用对应的文件 / 图像识别 Skills（你可以在之后的 Skills 章节节学会如何查找和启用这类 Skills）；
3. 得到一个初步列表后，再进一步过滤，例如：

```text
在上面这份候选发布票清单里，帮我筛出：
- 日期在 2025 年 10 月 1 日 到 12 月 31 日之间；
- 金额大于 200 元的发布票；
- 并按金额从大到小排序。
```

通过这样的方式，你可以把一件原本需要手工翻看半小时的事，压缩到几分钟甚至几十秒完成。

## 5.2 场景二：搭一个轻量级个人知识库

### 5.2.1 典型问题

- 收藏了很多技术文章节、公众号推文、PDF 报告，却很少回头看；
- 每次想起「之前好像看过一个 XXX 教程」，却找不到当时的链接；
- 想在某个主题上积累长期知识，但缺少一套趁手的工具。

### 5.2.2 基本思路

利用 OpenClaw 和相关 Skills，你可以：

- 把网页、PDF 等内容「剪藏」到某个知识库（本地或云端）；
- 自动生成摘要、标签、关键问题；
- 后续直接在会话中问：「关于 XXX，我之前都存了些什么？」。

### 5.2.3 示例工作流

1. 在浏览器中看到一篇好文章节，复制链接或主要内容；
2. 在 OpenClaw 会话中发布送类似指令：

```text
背景：我刚给你发了一篇关于「OpenClaw Skills 开发布」的文章节链接/内容。

目标：请帮我把这篇文章节存入一个名为「OpenClaw-学习笔记」的知识库中。

要求：
- 为这篇文章节生成一个简短标题；
- 输出 5 条以内的要点总结；
- 为将来检索方便，给出 3–5 个标签（tag），如：openclaw, skills, 开发布 等。

输出形式：
- 先给出标题；
- 再给出要点列表；
- 最后列出标签，并说明你打算如何在知识库中保存这些信息。
```

3. 当你积累了一批这样的条目之后，可以反向查询：

```text
请从「OpenClaw-学习笔记」这个知识库中，找出所有和「Skills 开发布」相关的条目，
并帮我整理成一份「Skills 开发布入门指南」，包括：
- 我需要先理解的 3 个概念；
- 官方文档中最重要的 3 个链接；
- 适合我先做的 2–3 个实战练习。
```

通过这种方式，你不再是「收藏一堆从不看」，而是逐步打造一个可问可用的个人第二大脑。

## 5.3 场景三：截图/文案 → 日程与提醒

### 5.3.1 典型问题

- 各种活动通知、会议安排都在微信群/飞书里发图或长文案；
- 每次要手动把时间、地点、主题抄到日历里，非常耗费注意力；
- 经常因此漏掉重要的会议或活动。

### 5.3.2 基本思路

结合 OpenClaw 的文本/图像理解能力 + 日历相关 Skills，可以做到：

- 把截图或文案转发/复制给 OpenClaw；
- 让它自动识别出时间、地点、参与人、主题等信息；
- 自动创建日历事件或待办，并与你的手机日历保持同步。

### 5.3.3 示例工作流

假设你在微信群看到一张活动海报截图，可以：\n\n1. 把这张图片发送/上传给 OpenClaw 对应的入口；\n2. 在会话中说明你的意图：\n\n```text
背景：我刚发给你一张活动海报截图，上面包含活动时间、地点和主题。

目标：请帮我从图片中提取关键信息，并创建一条日历事件。

具体要求：
- 如果时间只有日期没有具体时刻，默认开始时间为晚上 19:00，持续 2 小时；
- 地点如果过长，请压缩为简短形式；
- 事件标题使用「【活动】+ 主题」的形式；
- 备注中附上完整原文案/识别结果，方便我之后核对。

输出形式：
- 先以结构化方式列出【标题】【时间】【地点】【备注摘要】；
- 再调用日历相关的 Skills，实际创建事件（如果可能），或者给出可复制到日历 App 的文本。
```

对于纯文案通知（例如飞书群里的长消息），可以用类似模板，只是不再涉及图片上传。

## 5.4 小结：从「一次性使用」到「日常工作流」

这三个场景只是一个开始，重要的是你在过程中体会到：\n\n- OpenClaw 非常适合处理那种「人类觉得麻烦、但有规律可循」的任务；\n- 当你把「背景、目标、约束、输出形式」说清楚，它往往能给出不错的方案；\n- 这些「一次性的指令」完全可以逐步沉淀成固定模板，成为你的个人工作流。\n\n在后续章节里，我们会进一步介绍：\n\n- 如何用 Skills 让 OpenClaw 学会更多新本事；\n- 如何把这些场景串成自动化工作流，让它在**你没空管的时候也能继续帮你干活**。\n+
