# 第 2 章：选择你的上手路线

不同人的起点不一样：有人只是想先体验聊天，有人已经准备好买云服务器长期开机。

本章的目标，是帮你**选一条最合适自己、又不容易半途放弃的路线**。

---

## 2.1 三种典型读者画像

先看看你更像哪一类：

- **类型 A：只想先体验一下**
  - 现在只是好奇：「OpenClaw 到底和 ChatGPT 有啥不一样？」
  - 不想一上来就折腾云服务器、域名、HTTPS 这些东西。

- **类型 B：有 Mac，本地 Player**
  - 日常用 Mac 工作，愿意在本机上安装工具；
  - 希望本地文件、日历、备忘录等都能直接被 AI 使用；
  - 对隐私敏感，希望部分数据只留在本地。

- **类型 C：想 24 小时在线，云端 Player**
  - 已有或愿意购买一台云服务器（阿里云 / 腾讯云 / 火山引擎等）；
  - 希望手机、平板、任何地方都能随时连上 OpenClaw；
  - 未来可能给小团队一起用。

你可以不完全符合某一类，只要**大致接近**即可。

---

## 2.2 路线速览对比

| 路线         | 适合人群          | 部署位置 | 难度   | 成本（每月）     | 特点                       |
| ------------ | ----------------- | -------- | ------ | ---------------- | -------------------------- |
| 路线 A：体验 | 所有人，尤其是新手 | 本地     | ★☆☆    | 0 元             | 先感受交互，不追求 24 小时在线 |
| 路线 B：本地 | 有 Mac 的个人用户 | 本地     | ★★☆    | 0 元             | 体验最好，可用到系统级功能     |
| 路线 C：云端 | 想长期运行的人     | 云服务器 | ★★★（但可抄作业） | 约 10–50 元（看配置） | 24 小时在线，可多人共用       |

建议：

- 如果你**完全是新手**，可以从「路线 A → 路线 B 或 C」逐步升级；
- 如果你**已经有云服务器经验**（比如玩过自己的博客 / 面板），可以直接上「路线 C」。

## 2.3 路线 A：先体验聊天与基础功能

目标：**在不折腾复杂配置的前提下，感受一下 OpenClaw 的基本交互体验。**

典型步骤（高层概览，细节在第 3 章）：

1. 安装 OpenClaw（官方一行命令安装脚本）；
2. 使用默认或最简配置，启动 Gateway / Control UI；
3. 打开浏览器访问控制面板，发布送几条基础指令，例如：
   - 「请用中文介绍一下你是谁」
   - 「帮我列一个这周的待办清单」
   - 「假设你是我的个人助理，帮我规划一下明天的日程」

这一阶段不强求你立刻用到本地文件、Skills、自动化，只要：

- 确认环境装成功；
- 感受到「和 ChatGPT 不太一样」的那种感觉（例如可调用工具、管理会话）。

> 如果你在这一步都遇到了困难，可以直接跳去看第 8 章的「常见踩坑与排查」，我们会按错误信息分类给出解决办法。

## 2.4 路线 B：Mac 本地部署（强烈推荐有 Mac 的用户）

这条路线在官方教程和社区实践中被证明是**体验最好的路径之一**：

- 可以访问本地文件系统；
- 可以操作系统日历、备忘录等；
- 延迟低、隐私好。

你需要准备：

- 一台较新的 macOS 设备（建议 12+，Apple Silicon 更佳）；
- 稳定网络；
- 一个可用的大模型 API（可以是国内中转或国产模型）。

大致步骤：

1. 使用官方脚本在本地安装 OpenClaw；
2. 按照本书提供的「最小可用配置」模板，配置一个或两个模型提供商；
3. 启动本地 Gateway；
4. 在浏览器里打开 Control UI，开始对话；
5. 逐步接入：本地文件 → 日历 → 常用 Skills。

我们会在第 3 章给出**一步一步的命令和配置示例**，你只需要复制粘贴并根据注释修改自己的 API Key 即可。

## 2.5 路线 C：云端部署（24 小时在线）

如果你想要：

- 在手机、iPad、任意电脑上随时访问；
- 让 OpenClaw 帮你跑「定时任务」「网站监控」「自动推送消息」；
- 或者未来给团队成员一起使用；

那么非常推荐直接上云端部署。

你需要准备：

- 一台云服务器（2 核 4G 起步即可，新手可以选国内大厂的入门套餐）；
- 一个域名（可选，但强烈推荐，用于更友好的访问和 HTTPS）；
- 一点点 Linux 基础（至少会用 SSH 登录）。

大致步骤：

1. 在云服务器上用官方脚本或 Docker 安装 OpenClaw；
2. 配置基本防火墙、端口映射、安全组；
3. 配好网关访问地址（IP 或域名）；
4. 按照本书提供的模板写 `openclaw.json` 基础配置；
5. 启动 Gateway，并从本地浏览器/手机访问测试。

云端部署时，**安全和成本控制**尤其重要，我们会在后续章节节里专门讲：

- 如何用国产大模型（DeepSeek / Kimi 等）把成本压到几十元以内；
- 如何配置认证与访问控制，避免你的网关被乱用。
